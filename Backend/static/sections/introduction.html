Introduction

Ecommerce API provides RESTful endpoints for managing products, orders, users, and administrative functions. These APIs enable businesses to build e-commerce applications with features like product catalog management, order processing, user authentication, and administrative controls.

Base URL

http://localhost:8000/api/v1

Response Format

All successful responses follow a consistent structure:

List Endpoints

{
  "count": number,
  "data": array
}

Detail Endpoints

{
  "data": object
}

Create/Update Endpoints

{
  "data": object,
  "message": "string"
}

Error Responses

{
  "error": "string",
  "detail": "string"
}

Authentication

Most endpoints require authentication using JWT (JSON Web Tokens). Include the token in the Authorization header: 

Authorization: Bearer <your_access_token>

Status Codes

  • 200 OK - Request successful
  • 201 Created - Resource created successfully
  • 400 Bad Request - Invalid request data
  • 401 Unauthorized - Authentication required
  • 403 Forbidden - Insufficient permissions
  • 404 Not Found - Resource not found
  • 500 Internal Server Error - Server error
