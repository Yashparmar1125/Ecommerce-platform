Introduction

Ecommerce API provides RESTful endpoints for managing products, orders, users, and administrative functions. These APIs enable businesses to build e-commerce applications with features like product catalog management, order processing, user authentication, and administrative controls.

Base URL

http://localhost:8000/api/v1

Response Format

All successful responses follow a consistent structure:

List Endpoints

{
  "count": number,
  "data": array
}

Detail Endpoints

{
  "data": object
}

Error Responses

{
  "error": "string",
  "detail": "string"
}

Authentication

Most endpoints require authentication using JWT (JSON Web Tokens). Include the token in the Authorization header: 

Authorization: Bearer <your_access_token>

Authentication

POST /api/v1/users/register/

Register a new user account.

Request Body

{
  "email": "user@example.com",
  "password": "password123",
  "first_name": "John",
  "last_name": "Doe",
  "phone_number": "+1234567890"
}

Response

{
  "data": {
    "user": { ... },
    "tokens": {
      "access": "...",
      "refresh": "..."
    }
  },
  "message": "User registered successfully"
}
POST /api/v1/users/login/

Login with email and password.

Request Body

{
  "email": "user@example.com",
  "password": "password123"
}

Response

{
  "data": {
    "tokens": {
      "access": "...",
      "refresh": "..."
    }
  },
  "message": "Login successful"
}
POST /api/v1/users/token/refresh/

Refresh access token using refresh token.

Request Body

{
  "refresh": "your_refresh_token"
}

Response

{
  "data": {
    "access": "new_access_token",
    "refresh": "new_refresh_token"
  }
}
POST /api/v1/users/logout/

Logout and invalidate refresh token.

Request Body

{
  "refresh": "your_refresh_token"
}

Response

{
  "message": "Logged out successfully"
}

Users

GET /api/v1/users/me/

Get current user profile. Requires authentication.

Response

{
  "data": {
    "id": 1,
    "username": "user",
    "email": "user@example.com",
    "first_name": "John",
    "last_name": "Doe",
    "phone_number": "+1234567890"
  }
}
PATCH /api/v1/users/me/

Update current user profile. Requires authentication.

Addresses

GET /api/v1/users/addresses/

Get all addresses for current user. Requires authentication.

POST /api/v1/users/addresses/

Create a new address. Requires authentication.

Request Body

{
  "name": "Home",
  "street": "123 Main St",
  "city": "New York",
  "state": "NY",
  "zip_code": "10001",
  "phone": "+1234567890",
  "is_default": true
}
GET /api/v1/users/addresses/<id>/

Get a specific address. Requires authentication.

PATCH /api/v1/users/addresses/<id>/

Update an address. Requires authentication.

DELETE /api/v1/users/addresses/<id>/

Delete an address. Requires authentication.

Products

GET /api/v1/products/

Get all products. Public endpoint.

Query Parameters

  • featured - Filter featured products (true/false)
  • category - Filter by category name

Response

{
  "count": 10,
  "data": [
    {
      "id": 1,
      "name": "Product Name",
      "description": "...",
      "price": 99.99,
      "category": "Category",
      ...
    }
  ]
}
GET /api/v1/products/<id>/

Get product details with SKUs. Public endpoint.

Response

{
  "data": {
    "product": { ... },
    "skus": [ ... ]
  }
}
GET /api/v1/products/<id>/skus/

Get all SKUs for a product. Public endpoint.

GET /api/v1/products/search/

Search products. Public endpoint.

Query Parameters

  • q - Search query string
GET /api/v1/products/categories/

Get all categories. Public endpoint.

Orders

GET /api/v1/orders/

Get all orders for current user. Requires authentication.

Response

{
  "count": 5,
  "data": [
    {
      "id": 1,
      "total": 199.98,
      "status": "pending",
      "items": [ ... ],
      "address_details": { ... },
      "created_at": "2024-01-01T00:00:00Z"
    }
  ]
}
POST /api/v1/orders/

Create a new order. Requires authentication.

Request Body

{
  "address_id": 1,
  "items": [
    {
      "product_id": 1,
      "sku_id": 1,
      "quantity": 2
    },
    {
      "product_id": 2,
      "size": "M",
      "color": "Black",
      "quantity": 1
    }
  ]
}
GET /api/v1/orders/<id>/

Get order details. Requires authentication.
